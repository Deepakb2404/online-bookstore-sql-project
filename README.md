#  Online Book Store Database (SQL Project)

##  Project Overview

This project involves designing and analyzing a relational database for an Online Book Store using PostgreSQL. It focuses on managing book inventory, customer data, and order transactions while generating meaningful business insights using SQL.

##  Database Structure

The database consists of three main tables:

* **Books** – Stores book details (title, author, genre, price, stock)
* **Customers** – Stores customer information (name, email, location)
* **Orders** – Stores transaction details (order date, quantity, total amount)

All tables are connected using **Primary Keys and Foreign Keys** to maintain data integrity.

##  Key Features

* Relational database design with proper schema
* Data import using CSV files
* Use of constraints for accuracy and consistency
* Efficient query writing for data analysis

##  Key SQL Concepts Used

* JOINs (INNER JOIN, LEFT JOIN)
* Aggregate Functions (SUM, AVG, COUNT)
* GROUP BY & HAVING
* Filtering (WHERE)
* Sorting (ORDER BY, LIMIT)

##  Business Insights

* Total revenue generated from all orders
* Top-selling books and genres
* Customer purchase behavior and repeat customers
* High-value customers and their locations
* Inventory status and remaining stock after orders

##  Learning Outcome

* Improved database design and normalization skills
* Enhanced ability to write optimized SQL queries
* Gained practical experience in data analysis and business problem-solving

##  Tools & Technologies

* PostgreSQL
* pgAdmin
* SQL

##  Conclusion

This project demonstrates how SQL can be used to transform raw data into actionable insights, supporting business decisions in an e-commerce environment.
